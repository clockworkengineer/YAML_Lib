# YAML_Lib Public API

This document lists the stable public headers that are installed and intended for user consumption.

## Core library headers

- `YAML.hpp`
- `YAML_Core.hpp`
- `YAML_Config.hpp` (generated by CMake)

## Public interface headers

- `IAction.hpp`
- `ISource.hpp`
- `IDestination.hpp`
- `IParser.hpp`
- `IStringify.hpp`
- `ITranslator.hpp`
- `YAML_Interfaces.hpp`

## Public implementation headers used by the API

These headers are part of the public include chain and are installed because they are referenced by public headers:

- `YAML_Error.hpp`
- `YAML_Sources.hpp`
- `YAML_Destinations.hpp`
- `Default_Stringify.hpp`
- `Bencode_Stringify.hpp`
- `JSON_Stringify.hpp`
- `XML_Stringify.hpp`
- `Default_Parser.hpp`
- `YAML_Converter.hpp`
- `XML_Translator.hpp`
- `Default_Translator.hpp`

## Public node and variant headers

- `YAML_Hole.hpp`
- `YAML_Array.hpp`
- `YAML_Boolean.hpp`
- `YAML_Comment.hpp`
- `YAML_Document.hpp`
- `YAML_Null.hpp`
- `YAML_Number.hpp`
- `YAML_Dictionary.hpp`
- `YAML_String.hpp`
- `YAML_Node.hpp`
- `YAML_Node_Creation.hpp`
- `YAML_Node_Index.hpp`
- `YAML_Node_Reference.hpp`

## Notes

- `YAML_Impl.hpp` is intentionally not part of the installed public API.
- The installed headers are flattened into the `include/` install directory to support the current header-first public API design.
- Public headers are intended to be included via:

```cpp
#include "YAML.hpp"
#include "YAML_Core.hpp"
```

- Users should avoid including private implementation headers not listed here.
